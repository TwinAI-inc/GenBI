GenBI Chart Intelligence Pipeline

Concrete plan + 25 design questions to align on the smart chart generation system

Part 1: The Problem

Current Issues

  • Donut with 12 quarters — too many categories, nearly invisible segments. Should be a line chart or bar chart instead.
  • All bars — chart diversity is poor. Multiple bar charts when scatter, treemap, radar would be better.
  • No data preprocessing — LLM doesn't extract month from date, create bins, or derive boolean flags.
  • No chart suitability rules — donut with 12+ categories, bar with 50 bars, scatter with 2 identical columns.
  • Template-based, not intelligence-based — charts generated from column types, not from data patterns.
  • No Moiré prevention — too many bars causes visual interference.
  • Equal distribution detection — 25%/25%/25%/25% donut is useless, should be a table or nothing.

Part 2: Proposed Pipeline

Stage 1: Data Profiling NEW

  • Column classification — numeric, categorical, date, boolean, text, ordinal, ID
  • Distribution analysis — skewness, outliers, cardinality, null ratio per column
  • Relationship detection — correlation matrix between numeric columns
  • Feature derivation — extract month/year from dates, bin numeric into ranges, create True/False flags
  • Data summary — row count, column count, date range, unique values per categorical

Stage 2: Chart Candidate Generation IMPROVE

  • Rule engine — each chart type has preconditions (min/max categories, data types needed)
  • Insight scoring — rank candidates by information value (variance, correlation, trend strength)
  • Anti-patterns — reject donut with >8 categories, bar with >12 bars, scatter with identical columns
  • Diversity requirement — no two charts of same type, spread across chart families
  • Priority metric selection — most important metric first (revenue > count > rate)

Stage 3: Chart Rules & Guardrails NEW

  • Donut/Pie — max 6-8 segments, min segment >3%, no near-equal distributions
  • Bar chart — max 10-12 bars, sort descending, aggregate "Other" for rest
  • Line chart — needs time axis, min 4 data points, max 6 series
  • Scatter — needs 2 different numeric columns with correlation, min 20 data points
  • Heatmap — needs 2 categoricals + 1 numeric, max 10x10 grid
  • Treemap — needs hierarchical or size-based data, 3-15 items
  • Radar — needs 4-8 metrics on comparable scales
  • Map — needs geographic column (state/country names)
  • Gauge — needs rate/percentage/score metric
  • Funnel — needs sequential/stage data with decreasing values

Stage 4: Data Formatting NEW

  • Aggregation — auto-group by category, compute sum/avg/count as appropriate
  • Sorting — bars sorted by value, time series sorted chronologically
  • Binning — continuous data grouped into meaningful ranges
  • Truncation — top N items + "Other" bucket for long-tail distributions
  • Label formatting — shorten long labels, format currencies, format dates
  • Scale normalization — radar chart axes normalized to 0-100

Stage 5: AI Enhancement (Post-render) IMPROVE

  • Narrative captions — LLM generates insight text per chart
  • Anomaly detection — flag unusual data points
  • Suggested questions — generate relevant follow-up questions
  • Chart recommendations — suggest alternative views
